    Step 2 of 11

    Install — one package

    ChorusGraph is a native runtime (not a LangGraph wrapper). Python 3.11+. ChorusStack.defaults() attaches PrismCache automatically — no Redis required to start.

    $ pip install "chorusgraph==1.1.0" $ python -c "from chorusgraph import ChorusStack; print(ChorusStack.defaults().resolve_cache().name)" prism

    Swap later with .with_cache(RedisCacheBackend(...)) — same graph code. Optional: pip install "chorusgraph[gemini]" for live LLM graphs.

    Step 4 of 11

    Route Ledger — replay every hop

    Every node publishes artifacts with rule_chain and category. Cache decisions land here too (cache_hit, scores, duration).

    "steps": [ { "hop": "cache_gate", "cache_hit": true, "rule_chain": ["cache_gate=hit"], ... }, { "hop": "writer", "category_slug": "writer", ... } ]

    Production teams use this for “why did the agent say that?” — sovereignty-friendly, no third-party trace cloud required.

    Step 5 of 11

    Built-in semantic cache

    L1 PrismCache is on by default. cache_gate runs coarse 64-d recall → full verify. Toggle miss vs hit to see the product win.

    MISS: ingress → cache_gate → tools / ReAct → writer → seed
    HIT:  ingress → cache_gate → writer (skip LLM)

    Cold path — cache miss

    Empty cache → gate returns MISS → graph runs tools / agent → answer → seed PrismCache (quality-gated).

    $ chorusgraph-use-cases cache Turn 1 (cold): kind=miss is_hit=False -> continue graph: tools / ReAct / writer, then seed cache Seeded PrismCache for: 'What is the USD to EUR exchange rate today?'

    Warm path — cache hit

    Same (or seeded paraphrase) query → gate returns HIT with score → route straight to writer. ~ms latency, 0 LLM tokens on repeats.

    Turn 2 (repeat): kind=hit is_hit=True score=1.0 cached value: {'from_currency': 'USD', 'to_currency': 'EUR', 'rate': 0.92, ...} -> cache_gate routes straight to writer (skip LLM tools) Backend: prism (ChorusStack default)

    Default

    PrismCacheBackend — zero extra deps

    Swap

    RedisCacheBackend via with_cache()

    Step 6 of 11

    Warm chunks (L2) & PrismCortex (L3)

    L1 cache answers. L2 retrieves knowledge. L3 remembers user/session facts. Toggle to see the right recipe for each.

    Index once · warm at boot · query-only retrieve

    Do not re-encode the KB every turn. Use partitions so a catalog bump never invalidates kb_markdown. ADR-005.

    index(partition, version) → warm_retrieval → retrieve (query-only)
    rerank_policy="vectors_only" on PrismRAG
    $ chorusgraph-use-cases warm_chunks index kb_markdown=docs-v1, catalog=cat-v1 warm_retrieval → retrieval_ready=True catalog → cat-v2 (kb_markdown still docs-v1) retrieve handler rule_chain includes partition=kb_markdown

    Recall at ingress · digest off the hot path

    Wrong: user prefs in L1 whole-answer cache · sync digest before response · dump full transcript every hop.
    Right: recall_structured → confidence gate · schedule_digest async after the answer.

    ingress recall_structured → hops → response → async schedule_digest
    $ chorusgraph-use-cases cortex Turn 1: schedule_digest (non-blocking) Turn 2: memory_confidence=0.91 memory_recall=User prefers moderate risk Unrelated query: None (no fabricated fallback) Live: chorusgraph-finance-memory

    Warm: docs/ADR-005-warm-chunk-vectors.md · Cortex: chorusgraph-finance-memory / CortexMemoryBackend

    Step 8 of 11

    Multi-agent — role pipeline

    When the plan is a known multi-hop pipeline, use role-typed nodes on the native graph — not a dynamic Agent loop. Production FC2/HC2 add cache_gate before the role chain.

    START → researcher → writer → validator → END
    + cache: ingress → cache_gate -[hit]→ writer | -[miss]→ researcher → …

    Researcher

    Plans what to gather; publishes research_plan.

    Writer

    Drafts the answer from the plan artifact.

    Validator

    Checks the draft; ledger records ok=….

    $ chorusgraph-use-cases multi_agent Ledger path: researcher -> writer -> validator

    See docs/FINANCE_MULTIAGENT_CHORUS.md for FC2 wiring with cache hit skip.
